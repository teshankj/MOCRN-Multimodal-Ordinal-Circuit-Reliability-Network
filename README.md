# MOCRN: Physics-Guided Multimodal Neural Network for Analog Circuits

Official implementation of the paper: **"MOCRN-A Novel Physics-Guided Fault Modeling and Multimodal Neural Network for Fault Identification and Degradation Estimation in Analog Circuits"** published in *Circuits, Systems, and Signal Processing*.

MOCRN (Multimodal Ordinal Circuit Reliability Network) is a novel framework that combines physics-guided fault modeling with a multimodal neural network to identify faults and estimate degradation levels in analog circuits with high precision.



## 📖 Key Contributions

*   **Physics-Guided Modeling**: Integrates domain-specific circuit knowledge into the fault modeling process.
*   **Multimodal Fusion**: Utilizes both raw waveform data and engineered statistical features via a late-fusion architecture.
*   **Dual-Task Learning**: Simultaneously performs discrete **Fault Identification** and continuous **Degradation Estimation**.
*   **Ordinal Reliability**: Employs hybrid evidential ordinal loss to capture the natural progression of component degradation.

##  Getting Started

## 🛠️ Dataset Generation & Fault Injection

The dataset is generated by stepping through different **Fault Types (FT)** and **Degradation Levels (DEG)** using LTspice transient analysis.

###  Fault Injection Mechanism
Each circuit utilizes a structured parameter space to simulate progressive failure:
*   **Fault Type (FT)**: Defines the specific component or subsystem undergoing failure.
*   **Degradation (DEG)**: A scale from `0` (Healthy) to `100` (Degraded) that determines the severity of the fault.
*   **Deterministic Variation**: To ensure dataset diversity without sacrificing reproducibility, Monte Carlo (`mc()`) functions are replaced with deterministic trigonometric variations based on the `run` parameter (e.g., `0.1 * sin(run * 0.6283)`).

###  Circuit-Specific Fault Definitions

| Circuit | Fault Types (FT) | Key Components Affected |
| :--- | :--- | :--- |
| **Voltage Rectifier** | FT0–FT5 | Diode leakage, Load resistance, Source amplitude, Capacitor/ESR, Shunt path. |
| **Peak Detector** | FT0–FT4 | Hold capacitor leakage, Diode resistance, Discharge resistor drift, Series resistance. |
| **Diode Clamper** | FT0–FT4 | Clamping capacitor degradation, Diode ideality, Load resistor drift, Source degradation. |

###  Simulation Parameters
*   **Analysis Type**: Transient Analysis (`.tran`).
*   **Time Steps**: Typically `4ms` to `40ms` depending on the circuit frequency.
*   **Data Saving**: Critical nodes and currents are explicitly saved using `.save` commands to ensure high-fidelity feature extraction (e.g., `V(in)`, `V(out)`, `I(Rload)`).

## 🚀 How to Generate the Dataset

1.  **Open Source Files**: Load the `.asc` files using LTspice XVII or later.
2.  **Run Simulation**: Click the "Run" icon. LTspice will iterate through the `.step` parameters, generating waveforms for every combination of FT, DEG, and Run.
3.  **Export Data**: 
    *   After the simulation completes, select the waveform window.
    *   Go to **File > Export data as text**.
    *   Select the required traces (e.g., `V(in)`, `V(n1)`, etc.) and save the output as a `.txt` file in the `/data` folder.

###  Environment Setup
```powershell
# Create and activate virtual environment
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# Install requirements
pip install -r requirements.txt
```

###  Configuration
Update `config.yaml` to define your dataset locations and training parameters:
*   **`data.circuits`**: Paths to raw LTSpice output files.
*   **`training.`**: training parameters.

###  Execution
**Feature Importance Analysis**:
Run the consensus-based analysis to identify critical circuit features:
```powershell
python -m experiments.feature_analysis
```

**Model Training**:
Execute the physics-guided training pipeline:
```powershell
python -m experiments.train
```
